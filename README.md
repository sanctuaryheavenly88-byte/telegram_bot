# Telegram-бот для кафе

Асинхронный Telegram-бот на Python 3.11, aiogram 3, SQLAlchemy 2 и SQLite. Проект хранит меню, корзины, заказы, брони, категории и товары в базе данных. При первом запуске создаются таблицы и добавляются тестовые категории с товарами.

## Возможности

- Главное меню по `/start`.
- Меню кафе из базы данных: категории, товары, фото, описание и цена.
- Корзина: добавление, уменьшение/увеличение количества, удаление товара и очистка.
- Оформление заказа через FSM: самовывоз или доставка, имя, телефон, адрес, комментарий.
- Бронирование стола через FSM: имя, телефон, дата, время, количество гостей, комментарий.
- Информация о кафе из `.env`.
- История заказов пользователя.
- Админ-панель по `/admin` и кнопке в меню для ID из `ADMIN_IDS`.
- Уведомления администраторам о новых заказах и бронях.
- CRUD категорий и товаров внутри бота.

## Структура

```text
bot/
  handlers/

  database/
  states/

  main.py
  texts.py
```

## Как создать бота

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Укажите название и username бота.
4. Скопируйте токен и добавьте его в `.env` как `BOT_TOKEN`.

## Установка и запуск

```powershell
cd C:\Users\DE_DARK\cafe_telegram_bot
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
```

Заполните `.env`:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_IDS=123456789,987654321
DATABASE_URL=sqlite+aiosqlite:///./cafe.db
```

Чтобы узнать свой Telegram ID, можно написать боту `@userinfobot` или временно посмотреть `message.from_user.id` в логах.

Запуск:

```powershell
python -m bot.main
```

При первом запуске бот автоматически создаст таблицы SQLite и добавит тестовые категории: кофе, десерты и завтраки.

## Переход на PostgreSQL

Установите async-драйвер и замените `DATABASE_URL`:

```powershell
pip install asyncpg
```

```env
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/cafe_bot
```

Код CRUD и модели менять не нужно.

## Админ-панель

Администратор должен быть указан в `ADMIN_IDS`. После запуска доступны:

- `/admin` или кнопка `Админ-панель`.
- Просмотр новых заказов.
- Перевод статуса заказа по цепочке `новый` → `готовится` → `готов` → `выполнен`.
- Просмотр новых броней.
- Добавление, редактирование, удаление и скрытие категорий и товаров.

## Где менять тексты

Все основные строки интерфейса лежат в `bot/texts.py`, а данные о кафе берутся из `.env`.
